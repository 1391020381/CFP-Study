# 年金 (Annuity)

## 定义

**年金（Annuity）**是指**定期、等额的支付系列**。

简单来说，就是**每隔一段时间，固定投入或收取相同金额的钱**。

## 年金类型

### 1. 普通年金（Ordinary Annuity）
- **定义**：每期**期末**支付
- **最常见**：大多数养老金、年终奖、房贷还款
- **CFP考试默认**类型

### 2. 预付年金（Annuity Due）
- **定义**：每期**期初**支付
- **例子**：房租、保险费
- **特点**：比普通年金多计一期利息

### 3. 永续年金（Perpetuity）
- **定义**：永远持续支付，没有期限
- **例子**：英国永久债券、某些优先股
- **公式**：PV = PMT / r

## 计算公式

### 年金终值（计算到期总额）

```
FV = PMT × [((1+r)^n - 1) / r]
```

### 年金现值（计算当前价值）

```
PV = PMT × [1 - (1+r)^(-n)] / r
```

**其中**：
- **PMT** = 每期支付金额
- **r** = 每期利率
- **n** = 期数

## 实际例子

### 例1：教育储蓄（年金终值）
> **问题**：每年投5万，年化6%，投10年，到期有多少？
>
> **计算**：FV = 5 × [((1.06)^10 - 1) / 0.06] ≈ 65.9万
>
> **答案**：10年后约有65.9万

### 例2：养老金规划（年金现值）
> **问题**：退休后每年领6万，领20年，折现率4%，现在需要存多少？
>
> **计算**：PV = 6 × [1 - (1.04)^(-20)] / 0.04 ≈ 81.5万
>
> **答案**：现在需要约81.5万

### 例3：房贷计算（年金现值）
> **问题**：房贷100万，30年，月利率0.4%，每月还多少？
>
> 这是求PMT的问题：
> PMT = PV × [r / (1 - (1+r)^(-n))]
> PMT = 100 × [0.004 / (1 - 1.004^(-360))]
> PMT ≈ 0.51万 = 5,100元/月

## CFP 考试应用

| 应用场景 | 年金类型 | 说明 |
|----------|----------|------|
| **退休规划** | 年金现值 | 计算养老金需求 |
| **教育储蓄** | 年金终值 | 计算定期储蓄到期值 |
| **房贷计算** | 年金现值 | 计算每月还款额 |
| **保险评估** | 年金现值 | 评估保险产品价值 |

## 普通年金 vs 预付年金

| 类型 | 支付时点 | 终值关系 | 现值关系 |
|------|----------|----------|----------|
| **普通年金** | 期末 | 基准 | 基准 |
| **预付年金** | 期初 | × (1+r) | × (1+r) |

**预付年金 = 普通年金 × (1+r)**

## 常见错误

❌ **错误1**：混淆支付时点
- 正确：确认是期初还是期末

❌ **错误2**：利率和期数不匹配
- 正确：月还款要用月利率和月数

❌ **错误3**：混淆终值和现值
- 正确：终值是"到期有多少"，现值是"现在值多少"

## 快速记忆

```
定期等额支付 → 年金
期末支付    → 普通年金（默认）
期初支付    → 预付年金
永远支付    → 永续年金
```

## 关键要点

✅ **年金的核心是"定期、等额"**
✅ **CFP考试默认普通年金（期末支付）**
✅ **注意利率和期数的匹配**
✅ **区分求终值还是求现值**
